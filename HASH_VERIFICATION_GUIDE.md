# 🔐 Google IAP Ultimate - 文件哈希值验证指南

## 📋 概述

本指南说明如何使用文件哈希值来验证项目文件的完整性。哈希值可以用于：
- 验证文件是否被篡改
- 检查文件传输过程中的完整性
- 确保发布版本的可靠性

## 📁 哈希值文件

项目包含以下哈希值文件：

### 1. `project_hashes.json` - 详细哈希值文件
包含每个文件的详细信息：
- SHA256哈希值
- 文件大小
- 最后修改时间
- 文件路径

### 2. `file_hashes.txt` - 简化哈希值文件
仅包含文件路径和SHA256哈希值，便于快速验证。

## 🔧 使用方法

### 自动验证（推荐）
```bash
# 运行哈希值生成和验证工具
python tools/generate_hashes.py
```

### 手动验证
```bash
# 使用系统工具验证单个文件
certutil -hashfile "filename" SHA256

# 在Linux/macOS中使用
sha256sum "filename"
```

### 在线验证
您可以使用在线SHA256计算器验证文件哈希值。

## 🔒 安全建议

### 发布验证
在发布新版本时，请：
1. 重新生成哈希值文件
2. 验证所有文件的完整性
3. 将哈希值文件包含在发布包中

### 用户验证
用户下载项目后可以：
1. 运行验证工具检查文件完整性
2. 比较哈希值确保文件未被篡改
3. 报告任何哈希值不匹配的文件

## 📝 哈希值文件格式

### JSON格式 (`project_hashes.json`)
```json
{
  "project": "Google IAP Ultimate",
  "version": "6.0.0",
  "generated_at": "2024-01-01T12:00:00",
  "file_hashes": {
    "project.godot": {
      "size": 1024,
      "modified_time": "2024-01-01T10:00:00",
      "hash": "abc123..."
    }
  }
}
```

### 文本格式 (`file_hashes.txt`)
```
# Google IAP Ultimate - 文件哈希值
# 生成时间: 2024-01-01 12:00:00
# SHA256哈希值

abc123def456...  project.godot
def456ghi789...  README.md
```

## 🛠️ 开发工作流

### 1. 开发阶段
- 每次重要更改后重新生成哈希值
- 提交哈希值文件到版本控制
- 验证CI/CD流程中的文件完整性

### 2. 发布阶段
- 生成最终版本的哈希值
- 包含在发布说明中
- 提供给用户进行验证

### 3. 维护阶段
- 定期验证文件完整性
- 更新哈希值文件
- 监控文件变更
